Lab 2

Reflection

1. What's your modification? Any thoughts (why)? Steps?

My first edit was deleting the sky! Using the Magic Wand Selection tool in Pinta, I selected a blue pixel which, in turn, grabs all the neighbouring pixels of the same colour on my behalf. Then, I pressed the delete key. The sparsely populated sky pixels that remain are the pixels that make up the clouds.

Additionally, I copied the top portion of the evergreen tree, using the rectangular selection tool, and repurposed them as shrubs on the ground.

2. What are the differences you notice from the modification (e.g., file size change? Quality change? etc.)

When I saved my modified copy, I requested a lower-quality image. Despite reducing the file's size, I did not notice apparent quality differences.

Unsurprisingly, the modified image file is significantly smaller in length than the original image. The original image has a size of 3.8 MegaBytes; the modified image has a size of 1.8 MegaBytes.

3. Is there any special consideration when you created/ drew the new image(s)? What were the steps?

There were no special considerations when I created the new image as my image is not complex. My creation encompasses simple geometric shapes and text.

The following steps through the process on how to reproduce my clock:

  1. Outline a circle.
  2. Draw a smaller circle at the first circle's centroid.
  3. Write and arrange textualized numbers in the typical clock format.
  4. Draw two (2) rectangles, one longer in length than the other. These are the clock's hands.
  5. Attach one smaller edge of each retangular clock hand to the smaller circle from step 2.

4. Provide at least 5 guidelines to include images on the website.

  1. Favour alternative text over tooltips.
  2. Always include all relevant contextual information in the alt or an ARIA attribute.
  3. Images should adequatly contrast with the background.
  4. Nonfunctional images should either have an empty alt attribute or assume the "presentation" ARIA role.
  5. Images with longer explainations should not be embedding into any HTML property. Instead, the content should appear in an associated <figcapture> tag.
