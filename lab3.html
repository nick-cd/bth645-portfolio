1. The reason of choosing the file format when creating your audio

Here, I chose the MP3 file type. It's popular reputation made it the most portable, the driving factor for my decision.

2. The use of audio on your website regarding usability.

Playback controls

In every <audio> element, I specified the controls attribute to give the user the liberty to change playback as they wish. This design adheres to the W3C's audio standard with respect to control availability which suggests that recordings longer than three (3) seconds should have playback controls.

Contingency audio files types

Not every compression and decompression algorithms (codecs) are available on browsers. Striving for universal browser support, I provide multiple <source>s of the same recording within every <audio> tag.

4. Description of your editing on the file, e.g., cutting, assembly, fadein, fadeout, etc.

The steps below describe the procedure to recreate the manipulated recording.

  1. Select the time range from 10:54 to 11:54.
  2. Cut (Ctrl + X) the selection.
  3. Select the beginning of the recording.
  4. Paste (Ctrl + V) the selection.
  5. Select the time range from 0:00 (when the Tums commericial airs) to 0:30 (the transition between commericials).
  6. Copy (Ctrl + C) the selection.
  7. Repeact steps 3 and 4.
  8. Select the time range from 0:03.7 to 0:04.4.
  9. Copy (Ctrl + C) the seclection.
  10. Paste (Ctrl + V) the selection at time 0:04.4.
